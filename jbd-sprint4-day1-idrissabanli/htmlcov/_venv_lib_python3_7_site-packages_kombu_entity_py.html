39class Exchange(MaybeChannelBound): 

40 """An Exchange declaration. 

41 

42 Arguments: 

43 name (str): See :attr:`name`. 

44 type (str): See :attr:`type`. 

45 channel (kombu.Connection, ChannelT): See :attr:`channel`. 

46 durable (bool): See :attr:`durable`. 

47 auto_delete (bool): See :attr:`auto_delete`. 

48 delivery_mode (enum): See :attr:`delivery_mode`. 

49 arguments (Dict): See :attr:`arguments`. 

50 no_declare (bool): See :attr:`no_declare` 

51 

52 Attributes: 

53 name (str): Name of the exchange. 

54 Default is no name (the default exchange). 

55 

56 type (str): 

57 *This description of AMQP exchange types was shamelessly stolen 

58 from the blog post `AMQP in 10 minutes: Part 4`_ by 

59 Rajith Attapattu. Reading this article is recommended if you're 

60 new to amqp.* 

61 

62 "AMQP defines four default exchange types (routing algorithms) that 

63 covers most of the common messaging use cases. An AMQP broker can 

64 also define additional exchange types, so see your broker 

65 manual for more information about available exchange types. 

66 

67 * `direct` (*default*) 

68 

69 Direct match between the routing key in the message, 

70 and the routing criteria used when a queue is bound to 

71 this exchange. 

72 

73 * `topic` 

74 

75 Wildcard match between the routing key and the routing 

76 pattern specified in the exchange/queue binding. 

77 The routing key is treated as zero or more words delimited 

78 by `"."` and supports special wildcard characters. `"*"` 

79 matches a single word and `"#"` matches zero or more words. 

80 

81 * `fanout` 

82 

83 Queues are bound to this exchange with no arguments. Hence 

84 any message sent to this exchange will be forwarded to all 

85 queues bound to this exchange. 

86 

87 * `headers` 

88 

89 Queues are bound to this exchange with a table of arguments 

90 containing headers and values (optional). A special 

91 argument named "x-match" determines the matching algorithm, 

92 where `"all"` implies an `AND` (all pairs must match) and 

93 `"any"` implies `OR` (at least one pair must match). 

94 

95 :attr:`arguments` is used to specify the arguments. 

96 

97 

98 .. _`AMQP in 10 minutes: Part 4`: 

99 https://bit.ly/2rcICv5 

100 

101 channel (ChannelT): The channel the exchange is bound to (if bound). 

102 

103 durable (bool): Durable exchanges remain active when a server restarts. 

104 Non-durable exchanges (transient exchanges) are purged when a 

105 server restarts. Default is :const:`True`. 

106 

107 auto_delete (bool): If set, the exchange is deleted when all queues 

108 have finished using it. Default is :const:`False`. 

109 

110 delivery_mode (enum): The default delivery mode used for messages. 

111 The value is an integer, or alias string. 

112 

113 * 1 or `"transient"` 

114 

115 The message is transient. Which means it is stored in 

116 memory only, and is lost if the server dies or restarts. 

117 

118 * 2 or "persistent" (*default*) 

119 The message is persistent. Which means the message is 

120 stored both in-memory, and on disk, and therefore 

121 preserved if the server dies or restarts. 

122 

123 The default value is 2 (persistent). 

124 

125 arguments (Dict): Additional arguments to specify when the exchange 

126 is declared. 

127 

128 no_declare (bool): Never declare this exchange 

129 (:meth:`declare` does nothing). 

130 """

373class Queue(MaybeChannelBound): 

374 """A Queue declaration. 

375 

376 Arguments: 

377 name (str): See :attr:`name`. 

378 exchange (Exchange, str): See :attr:`exchange`. 

379 routing_key (str): See :attr:`routing_key`. 

380 channel (kombu.Connection, ChannelT): See :attr:`channel`. 

381 durable (bool): See :attr:`durable`. 

382 exclusive (bool): See :attr:`exclusive`. 

383 auto_delete (bool): See :attr:`auto_delete`. 

384 queue_arguments (Dict): See :attr:`queue_arguments`. 

385 binding_arguments (Dict): See :attr:`binding_arguments`. 

386 consumer_arguments (Dict): See :attr:`consumer_arguments`. 

387 no_declare (bool): See :attr:`no_declare`. 

388 on_declared (Callable): See :attr:`on_declared`. 

389 expires (float): See :attr:`expires`. 

390 message_ttl (float): See :attr:`message_ttl`. 

391 max_length (int): See :attr:`max_length`. 

392 max_length_bytes (int): See :attr:`max_length_bytes`. 

393 max_priority (int): See :attr:`max_priority`. 

394 

395 Attributes: 

396 name (str): Name of the queue. 

397 Default is no name (default queue destination). 

398 

399 exchange (Exchange): The :class:`Exchange` the queue binds to. 

400 

401 routing_key (str): The routing key (if any), also called *binding key*. 

402 

403 The interpretation of the routing key depends on 

404 the :attr:`Exchange.type`. 

405 

406 * direct exchange 

407 

408 Matches if the routing key property of the message and 

409 the :attr:`routing_key` attribute are identical. 

410 

411 * fanout exchange 

412 

413 Always matches, even if the binding does not have a key. 

414 

415 * topic exchange 

416 

417 Matches the routing key property of the message by a primitive 

418 pattern matching scheme. The message routing key then consists 

419 of words separated by dots (`"."`, like domain names), and 

420 two special characters are available; star (`"*"`) and hash 

421 (`"#"`). The star matches any word, and the hash matches 

422 zero or more words. For example `"*.stock.#"` matches the 

423 routing keys `"usd.stock"` and `"eur.stock.db"` but not 

424 `"stock.nasdaq"`. 

425 

426 channel (ChannelT): The channel the Queue is bound to (if bound). 

427 

428 durable (bool): Durable queues remain active when a server restarts. 

429 Non-durable queues (transient queues) are purged if/when 

430 a server restarts. 

431 Note that durable queues do not necessarily hold persistent 

432 messages, although it does not make sense to send 

433 persistent messages to a transient queue. 

434 

435 Default is :const:`True`. 

436 

437 exclusive (bool): Exclusive queues may only be consumed from by the 

438 current connection. Setting the 'exclusive' flag 

439 always implies 'auto-delete'. 

440 

441 Default is :const:`False`. 

442 

443 auto_delete (bool): If set, the queue is deleted when all consumers 

444 have finished using it. Last consumer can be canceled 

445 either explicitly or because its channel is closed. If 

446 there was no consumer ever on the queue, it won't be 

447 deleted. 

448 

449 expires (float): Set the expiry time (in seconds) for when this 

450 queue should expire. 

451 

452 The expiry time decides how long the queue can stay unused 

453 before it's automatically deleted. 

454 *Unused* means the queue has no consumers, the queue has not been 

455 redeclared, and ``Queue.get`` has not been invoked for a duration 

456 of at least the expiration period. 

457 

458 See https://www.rabbitmq.com/ttl.html#queue-ttl 

459 

460 **RabbitMQ extension**: Only available when using RabbitMQ. 

461 

462 message_ttl (float): Message time to live in seconds. 

463 

464 This setting controls how long messages can stay in the queue 

465 unconsumed. If the expiry time passes before a message consumer 

466 has received the message, the message is deleted and no consumer 

467 will see the message. 

468 

469 See https://www.rabbitmq.com/ttl.html#per-queue-message-ttl 

470 

471 **RabbitMQ extension**: Only available when using RabbitMQ. 

472 

473 max_length (int): Set the maximum number of messages that the 

474 queue can hold. 

475 

476 If the number of messages in the queue size exceeds this limit, 

477 new messages will be dropped (or dead-lettered if a dead letter 

478 exchange is active). 

479 

480 See https://www.rabbitmq.com/maxlength.html 

481 

482 **RabbitMQ extension**: Only available when using RabbitMQ. 

483 

484 max_length_bytes (int): Set the max size (in bytes) for the total 

485 of messages in the queue. 

486 

487 If the total size of all the messages in the queue exceeds this 

488 limit, new messages will be dropped (or dead-lettered if a dead 

489 letter exchange is active). 

490 

491 **RabbitMQ extension**: Only available when using RabbitMQ. 

492 

493 max_priority (int): Set the highest priority number for this queue. 

494 

495 For example if the value is 10, then messages can delivered to 

496 this queue can have a ``priority`` value between 0 and 10, 

497 where 10 is the highest priority. 

498 

499 RabbitMQ queues without a max priority set will ignore 

500 the priority field in the message, so if you want priorities 

501 you need to set the max priority field to declare the queue 

502 as a priority queue. 

503 

504 **RabbitMQ extension**: Only available when using RabbitMQ. 

505 

506 queue_arguments (Dict): Additional arguments used when declaring 

507 the queue. Can be used to to set the arguments value 

508 for RabbitMQ/AMQP's ``queue.declare``. 

509 

510 binding_arguments (Dict): Additional arguments used when binding 

511 the queue. Can be used to to set the arguments value 

512 for RabbitMQ/AMQP's ``queue.declare``. 

513 

514 consumer_arguments (Dict): Additional arguments used when consuming 

515 from this queue. Can be used to to set the arguments value 

516 for RabbitMQ/AMQP's ``basic.consume``. 

517 

518 alias (str): Unused in Kombu, but applications can take advantage 

519 of this, for example to give alternate names to queues with 

520 automatically generated queue names. 

521 

522 on_declared (Callable): Optional callback to be applied when the 

523 queue has been declared (the ``queue_declare`` operation is 

524 complete). This must be a function with a signature that 

525 accepts at least 3 positional arguments: 

526 ``(name, messages, consumers)``. 

527 

528 no_declare (bool): Never declare this queue, nor related 

529 entities (:meth:`declare` does nothing).

677 def get(self, no_ack=None, accept=None): 

678 """Poll the server for a new message. 

679 

680 This method provides direct access to the messages in a 

681 queue using a synchronous dialogue, designed for 

682 specific types of applications where synchronous functionality 

683 is more important than performance. 

684 

685 Returns: 

686 ~kombu.Message: if a message was available, 

687 or :const:`None` otherwise. 

688 

689 Arguments: 

690 no_ack (bool): If enabled the broker will 

691 automatically ack messages. 

692 accept (Set[str]): Custom list of accepted content types. 

693 """ 

694 no_ack = self.no_ack if no_ack is None else no_ack 

695 message = self.channel.basic_get(queue=self.name, no_ack=no_ack) 

696 if message is not None:
